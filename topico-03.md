1) Banco de dados: É uma coleção organizada de dados relacionados, armazenados e acessados ​​eletronicamente. Os bancos de dados são projetados para armazenar informações de forma estruturada, permitindo a recuperação eficiente e a manipulação dos dados.

2) Dado: É uma representação simbólica de fatos, conceitos ou instruções em uma forma adequada para processamento. Os dados podem ser numéricos, textuais, de áudio, vídeo, entre outros.

3) Tipo de dado: É uma classificação que define o formato e as operações possíveis em um dado. Exemplos de tipos de dados incluem inteiros, caracteres, datas, booleanos, entre outros.

4) Meta-dado: São dados que descrevem ou fornecem informações sobre outros dados. Eles descrevem a estrutura, definição, relacionamentos e restrições dos dados em um banco de dados.

5) Tamanho e complexidade do banco de dados: Refere-se à quantidade de dados armazenados em um banco de dados e à complexidade das estruturas e relacionamentos entre esses dados. Quanto maior e mais complexo o banco de dados, maior será o desafio de gerenciá-lo e garantir sua eficiência.

6) Banco de dados mantido manualmente ou informatizado: Refere-se à forma como os dados são armazenados e gerenciados. Um banco de dados mantido manualmente envolve o uso de arquivos físicos e métodos manuais para acessar e manipular os dados. Já um banco de dados informatizado é gerenciado por um Sistema de Gerenciador de Banco de Dados (SGBD), que automatiza as tarefas de armazenamento, recuperação e manipulação dos dados.

7) Sistema de gerenciador de banco de dados (SGBD): É um software que permite criar, gerenciar e manipular bancos de dados. Ele fornece uma interface para os usuários interagirem com os dados, realizando operações como inserção, atualização, exclusão e consulta.

8) Definição, construção, manipulação de um banco de dados: Refere-se às etapas envolvidas no ciclo de vida de um banco de dados. A definição envolve a especificação da estrutura, tipos de dados e restrições do banco de dados. A construção envolve a criação física do banco de dados, incluindo tabelas, índices e outras estruturas. A manipulação envolve a inserção, atualização, exclusão e recuperação dos dados no banco de dados.

9) Programa de aplicação: É um software desenvolvido para realizar tarefas específicas dentro de um banco de dados. Os programas de aplicação interagem com o SGBD para acessar e manipular os dados de acordo com as necessidades do usuário.

10) Consulta: É uma solicitação de informações ou instruções enviada a um banco de dados para recuperar dados específicos que atendam a certos critérios. Uma consulta pode ser usada para buscar dados, filtrar informações, calcular valores agregados ou realizar operações complexas nos dados armazenados no banco de dados.

11) Transação: É uma unidade lógica de trabalho que consiste em uma sequência de operações de banco de dados (como inserção, atualização ou exclusão) que devem ser executadas de forma atômica, consistente, isolada e durável. Uma transação garante a integridade dos dados e a recuperação em caso de falhas.

12) Sistema de banco de dados: É um conjunto de componentes inter-relacionados que trabalham juntos para gerenciar um banco de dados. Isso inclui o software do SGBD, o hardware onde o banco de dados está armazenado, os usuários e as aplicações que interagem com o banco de dados e os procedimentos para operar e manter o sistema.

13) Especificação de requisitos e análise conceitual: É a etapa inicial do projeto de um banco de dados, onde os requisitos do sistema são identificados e analisados. Isso envolve entender as necessidades dos usuários e definir os objetivos e restrições do banco de dados. A análise conceitual resulta na criação de um modelo conceitual que representa as entidades, relacionamentos e restrições do domínio do problema.

14) Projeto conceitual: É a etapa em que o modelo conceitual, derivado da análise conceitual, é transformado em um esquema de banco de dados conceitual. Nessa fase, são definidas as tabelas, atributos e relacionamentos do banco de dados, sem se preocupar com os detalhes de implementação.

15) Projeto lógico: É a etapa em que o esquema de banco de dados conceitual é transformado em um esquema de banco de dados lógico, que representa a estrutura de armazenamento e os relacionamentos entre os dados no nível lógico. Nessa fase, são tomadas decisões sobre a organização física dos dados, como a definição de chaves primárias, índices e estruturas de armazenamento.

16) Projeto físico: É a etapa em que o esquema de banco de dados lógico é transformado em um esquema de banco de dados físico, que representa a implementação real do banco de dados no nível físico. Nessa fase, são tomadas decisões sobre a alocação de espaço em disco, a definição de estruturas de armazenamento e a otimização do desempenho.

17) Abordagem tradicional de arquivos X abordagem de banco de dados: A abordagem tradicional de arquivos envolve o armazenamento e gerenciamento de dados em arquivos separados, onde cada aplicação tem seus próprios arquivos de dados e lógica de acesso. Já a abordagem de banco de dados utiliza um SGBD para armazenar e gerenciar os dados de forma centralizada, permitindo o compartilhamento e a integração dos dados entre diferentes aplicações.

18) Natureza auto-descritiva de um sistema de banco de dados: Refere-se à capacidade de um sistema de banco de dados de armazenar informações sobre sua própria estrutura e características. Um sistema de banco de dados contém metadados que descrevem a definição do banco de dados, como tabelas, atributos, relacionamentos e restrições. Isso permite que o sistema de banco de dados entenda e manipule os dados de maneira eficiente.

19) Isolamento entre programas e dados e abstração de dados: Um sistema de banco de dados fornece isolamento entre programas e dados, o que significa que as aplicações podem acessar os dados sem precisar conhecer os detalhes de implementação. Além disso, um sistema de banco de dados oferece abstração de dados, que permite que os usuários e aplicações vejam os dados em um nível conceitual mais alto, ocultando os detalhes de armazenamento e organização física.

20) Suporte a múltiplas visões dos dados: Um sistema de banco de dados permite que diferentes usuários tenham visões personalizadas dos dados. Cada usuário pode ter uma visão específica dos dados, definida por consultas e restrições personalizadas. Isso permite que os usuários vejam e manipulem os dados de acordo com suas necessidades específicas, enquanto o sistema de banco de dados garante a integridade e consistência dos dados subjacentes.

21) Compartilhamento de dados e processamento de transações multiusuário: Refere-se à capacidade do sistema de banco de dados de permitir que vários usuários acessem e modifiquem os dados simultaneamente. O compartilhamento de dados garante que os usuários tenham acesso aos dados atualizados e consistentes, enquanto o processamento de transações multiusuário garante a execução correta e concorrente das transações realizadas por diferentes usuários.

22) Administradores de banco de dados: São responsáveis pela administração e gerenciamento do sistema de banco de dados. Eles são encarregados de projetar, implementar, monitorar e manter o banco de dados, garantindo a segurança, integridade e disponibilidade dos dados. Os administradores de banco de dados também são responsáveis por realizar backups, otimizar o desempenho e solucionar problemas no sistema.

23) Designers de banco de dados: São responsáveis pelo projeto e modelagem do banco de dados. Eles analisam os requisitos do sistema, identificam as entidades, relacionamentos e restrições envolvidas e criam o esquema de banco de dados conceitual, lógico e físico. Os designers de banco de dados também devem considerar aspectos de desempenho, segurança e escalabilidade ao projetar o banco de dados.

24) Usuários finais: São os usuários que interagem diretamente com o sistema de banco de dados para realizar tarefas específicas. Eles podem ser usuários comuns que consultam e atualizam os dados, ou usuários especializados que utilizam recursos avançados do sistema de banco de dados.

25) Classificação de usuários finais (casuais, paramétricos, sofisticados, autônomos): Refere-se à classificação dos usuários finais com base em seu nível de conhecimento e habilidades no uso do sistema de banco de dados. Os usuários casuais são aqueles que têm conhecimentos básicos e realizam tarefas simples de consulta e atualização. Os usuários paramétricos têm conhecimentos intermediários e podem realizar consultas mais complexas e personalizadas. Os usuários sofisticados têm conhecimentos avançados e podem criar consultas complexas, realizar análises e criar relatórios personalizados. Os usuários autônomos têm conhecimentos especializados e são capazes de projetar e implementar seus próprios bancos de dados.

26) Analistas de sistema e programadores de aplicativos (engenheiros de software): São profissionais responsáveis por desenvolver e manter os aplicativos que interagem com o sistema de banco de dados. Os analistas de sistema são responsáveis por analisar os requisitos dos usuários e projetar a lógica de negócios dos aplicativos. Os programadores de aplicativos são responsáveis por implementar essa lógica em linguagens de programação e integrá-la com o sistema de banco de dados.

27) Redundância: Refere-se à duplicação desnecessária de dados em um banco de dados. A redundância pode ocorrer quando os mesmos dados são armazenados em várias tabelas ou quando os dados são repetidos em diferentes registros. A redundância pode levar a problemas de integridade, consistência e desempenho do banco de dados.

28) Redundância controlada: Refere-se à redundância intencionalmente introduzida no projeto de banco de dados para melhorar o desempenho ou a disponibilidade dos dados. A redundância controlada envolve a cópia seletiva de determinados dados em locais diferentes para permitir um acesso mais rápido ou fornecer uma cópia de backup em caso de falhas.

29) Restrição de acesso não autorizado: Refere-se à proteção dos dados contra acesso não autorizado. É importante que o sistema de banco de dados implemente mecanismos de segurança, como autenticação e autorização, para controlar quem pode acessar e modificar os dados. Restrições de acesso não autorizado garantem a confidencialidade e a integridade dos dados, evitando acessos indevidos ou alterações não autorizadas.

30) Persistência de dados: Refere-se à capacidade de armazenar os dados de forma duradoura, mesmo após o encerramento do sistema. Isso envolve técnicas de armazenamento em disco ou em outros meios de armazenamento.

31) Processamento de consulta: É o processo de executar uma consulta em um banco de dados para recuperar informações específicas. Isso envolve a análise da consulta, a otimização do plano de execução e a busca dos dados relevantes.

32) Restaurar e recuperar: Refere-se à capacidade de restaurar um banco de dados a partir de um estado anterior após uma falha ou erro. Isso envolve a recuperação dos dados a partir de backups ou logs de transações.

33) Restrições de integridade: São regras definidas para garantir a integridade dos dados em um banco de dados. Isso inclui restrições como chaves primárias, chaves estrangeiras, restrições de valor mínimo/máximo, entre outros.

34) Integridade referencial: É um tipo específico de restrição de integridade que garante que as relações entre as tabelas sejam mantidas corretamente. Por exemplo, uma chave estrangeira garante que os valores em uma coluna correspondam aos valores em outra tabela.

35) Regras de negócio: São regras definidas para refletir as políticas e procedimentos específicos de uma organização. Elas são aplicadas aos dados e ajudam a garantir que o banco de dados atenda às necessidades e requisitos da organização.

36) Modelos de dados, esquemas e instâncias: Modelos de dados são representações abstratas dos dados e relacionamentos em um banco de dados. Esquemas são descrições lógicas do banco de dados, incluindo tabelas, colunas e restrições. Instâncias são os dados reais armazenados no banco de dados.

37) Esquema interno, esquema conceitual, esquemas externos: Esquema interno refere-se à representação interna do banco de dados, incluindo a estrutura física e o armazenamento dos dados. Esquema conceitual é a visão global do banco de dados, descrevendo as tabelas, relacionamentos e restrições. Esquemas externos são visões específicas do banco de dados para usuários finais ou grupos de usuários.

38) Linguagens de banco de dados: São linguagens específicas usadas para interagir com bancos de dados. Elas incluem linguagens de consulta, como SQL, e linguagens de programação dedicadas à manipulação de dados em bancos de dados.

39) SQL: Structured Query Language (SQL) é uma linguagem de consulta padrão usada para interagir com bancos de dados relacionais. Ela permite realizar operações como consultas, inserções, atualizações e exclusões de dados em um banco de dados.

40) Abstração de dados: É a ideia de ocultar detalhes complexos do armazenamento e manipulação de dados, fornecendo uma visão simplificada e organizada dos dados. Isso permite que os usuários interajam com os dados de forma mais fácil e eficiente.

41) Modelo entidade relacionamento: É um modelo conceitual usado para representar e descrever a estrutura de um banco de dados. Ele utiliza entidades para representar objetos do mundo real, relacionamentos para mostrar as conexões entre as entidades e atributos para descrever as características das entidades.

42) Modelo relacional: É um modelo de dados que organiza os dados em tabelas, onde cada tabela representa uma entidade e as relações são estabelecidas por meio de chaves primárias e chaves estrangeiras. É amplamente utilizado em sistemas de gerenciamento de bancos de dados relacionais.

43) Independência de dados: Refere-se à capacidade de alterar a estrutura ou organização interna de um banco de dados sem afetar os programas ou aplicativos que o utilizam. Isso permite que as alterações sejam feitas no banco de dados sem interromper as operações e funcionalidades existentes.

44) Módulos que compõem um SGBD: Um Sistema de Gerenciamento de Banco de Dados (SGBD) é composto por vários módulos que desempenham funções específicas. Alguns dos principais módulos são: o Módulo de Armazenamento, responsável pelo gerenciamento físico do armazenamento dos dados; o Módulo de Consulta e Processamento, responsável pela execução de consultas e operações nos dados; o Módulo de Segurança, responsável pela proteção dos dados e controle de acesso; e o Módulo de Controle de Transações, responsável pelo gerenciamento das transações no banco de dados. Cada módulo desempenha um papel fundamental no funcionamento e na eficiência do SGBD.
